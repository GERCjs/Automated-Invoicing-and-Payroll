## Goal (supervisor requirement)
Payment record, refund record, invoice status, invoice PDF AND Excel export must all show the same refund information. Every successful refund creates a `PaymentRefund` record; totals shown in exports are derived from succeeded `PaymentRefund` records.

## Failing test

Failure: `payment_summary["refunded_amount"]` is `0.00`, expected `109.00`.

Production code (`_resolve_invoice_payment_summary`, invoicing/exports.py:129, refund summing ~:140-147) correctly derives refunded totals only from SUCCEEDED `PaymentRefund` records — this is the desired canonical behavior (the sibling test at :2386 asserts the "no matching refund record" case is flagged with a data-issue message, and must keep passing).
